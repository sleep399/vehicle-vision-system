# 全局错误码表

本项目当前采用 FastAPI 默认错误响应，不额外在每个业务接口中重复定义错误结构。业务接口发生 HTTP 错误时统一返回：

```json
{"detail":"错误说明"}
```

请求参数校验失败时返回：

```json
{"detail":[{"loc":["body","field"],"msg":"...","type":"..."}]}
```

| HTTP 状态 | 全局错误码 | 含义 | 典型触发场景 |
| --- | --- | --- | --- |
| 400 | `BAD_REQUEST` | 请求合法性或业务前置条件不满足 | 重复用户名、验证码无效或过期、验证码目标类型非法 |
| 401 | `UNAUTHORIZED` | 缺少、过期或无效的 Bearer Token | 调用 `/api/auth/me` 未登录 |
| 403 | `FORBIDDEN` | 已认证主体无权继续操作 | 已禁用用户进行验证码登录 |
| 404 | `NOT_FOUND` | 请求的资源不存在 | 微信扫码会话不存在 |
| 422 | `VALIDATION_ERROR` | 路径、查询参数、请求体或表单校验失败 | 缺少 `file`、`interval` 不在 1–60、JSON 字段类型错误 |
| 500 | `RECOGNITION_FAILURE` | 图像解析、模型推理或识别过程异常 | 图片损坏、模型文件不可用、推理依赖异常 |
| 500 | `INTERNAL_ERROR` | 未分类的服务内部异常 | 告警智能体或第三方服务调用失败 |

WebSocket 不使用 HTTP 状态体。建立连接后若发生业务异常，服务端发送：

```json
{"type":"error","message":"错误说明"}
```

说明：`BAD_REQUEST` 等为本文档的全局语义标识，当前运行时响应只返回 HTTP 状态和 `detail`。若后续需要机器可读业务码，应通过统一异常处理器将响应升级为 `{ "code", "message", "request_id" }`，并保持本表为唯一维护来源。
